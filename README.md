## Running a project
If you want to run or build it yourself, you can use this [demo project](https://github.com/LouisH96/Tower). It's a very simple project to use and test some of the features in this project.

## Experimental Context
* **Constant work in progress:** Most features aren't really "finished". Features only get added or refined as the need arises.
* **Lacks typical engine features:** Since I'm currently only using the engine for smaller projects, there was no need yet for typical engine features like Scene-Graphs, Entity-Component-System or Memory Pools.\
However, I have implemented features like these before in school projects, such as this [2D Engine](https://github.com/LouisH96/Minigin).
* **Compile time focus:**  I initially focused on keeping calculations at compile-time where possible. Making me focus on better design and metaprogramming instead of filling the gaps with runtime calculations.\
  Admittedly, this is somewhat contrary to the nature of modern game engines, which are these large, dynamic systems designed for games or mods to be "plugged" into.
* **Solo Development:** Since this never was going to be a collaborative project, I experimented with different coding styles and didn't spend much time on writing comments and documentation.
 Consequently, the project lacks the consistency and documentation standards I do maintain in collaborative projects.

In collaborative or professional projects, I strictly follow best practices and guidelines; however, this leaves/left no room to try out alternatives that might be weaker.
That's why this project is the ideal place to explore other approaches.
Since this is a hobby project, I like to compare it with painting for fun: you shouldn't worry about what's popular, which paint lasts the longest, or what sells best,
you should just paint how and what you want! :paintbrush:
